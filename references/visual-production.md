# 横屏视频素材制作手册

当用户提供成稿文案后，读取本文件，输出素材资源、Manim 原理动画提示词、Remotion 动效提示词和 AI 视频生成提示词。目标是帮用户直接去找素材、丢给视频 agent、最后在剪映/CapCut 拼接。

## 输出原则

- 不写文案，只把用户文案拆成可剪辑的镜头。
- 不输出选择题，直接给推荐执行方案。
- 每段口播都要说明放几段视频，每段视频多长。
- 每段视频只标一个主要类别：`网上素材`、`Manim 原理动画`、`Remotion 动效`、`AI 视频`。
- 每段视频都要写清：画面内容、镜头运动、搜索词或生成提示词、剪辑用途。
- 默认横屏 16:9，1920x1080，30fps。
- 默认按手机观看检查可读性：横屏视频在手机上会被缩小，关键元素必须大、清楚、集中。

## 手机观看可读性规则

虽然成片是横屏 16:9，但大量观众会用手机看。规划每个镜头时都要按“小屏一眼看懂”处理：

- 主体产品、人物、芯片、图表核心、数字或排名必须占据画面明显比例；关键主体建议至少占画面宽度的 30%-45%。
- 不安排密集小字、复杂说明框、细碎图标、拥挤 UI 截图、远景小物体或看不清的发布会大屏。
- 网上素材优先选近景、特写、清晰 Logo、单一主体、慢速镜头；避免全景里主体太小。
- Manim/Remotion 中，关键文字、数字和标签必须足够大；线条、箭头、节点不能细到手机上丢失。
- 来源角标可以小，但仍要可读；如果来源太长，缩成机构名或短域名。
- 一个镜头只服务一个视觉重点，不要同时塞满产品、图表、长标题、多个 Logo 和大量标签。

## 文案拆段方法

若用户文案没有时间码，按口播速度估算：

- 快节奏中文口播：每秒约 5-7 个汉字。
- 信息密集段：每 8-14 秒切成一个主体段。
- 榜单文案：按 No.5 到 No.1 或自然段拆。
- 科普文案：按“问题/原理/应用/冲击/结论”拆。
- 历史盘点：按时间节点或发明对象拆。

不要改写用户文案，只做：

- 分段标题
- 时间估算
- 画面对应
- 必要时标注“这一句需要事实核查”

## 视频素材拆分规则

按用户文案结构拆，不按固定五段硬拆：

- 榜单文案：每个 `No.x` 就是一个制作部分。
- 非榜单文案：按用户已有小标题、自然段或语义节奏拆分。
- 开头和结尾需要单独规划为制作部分；它们不算入任何一个 `No.x` 的 8-12 个画面。
- 每个 `No.x` 或主要制作部分默认安排 8-12 个画面。
- 开头部分默认安排 2-4 个画面，用于建立选题冲击、标题识别和第一秒注意力。
- 结尾部分默认安排 1-3 个画面，用于收束观点、强化最后一句和留出剪映字幕/账号信息空间。
- 每个画面只标一个主要类别：`网上素材`、`Manim 原理动画`、`Remotion 动效`、`AI 视频`。
- 科技原理、科学机制、数据变化、时间轴或结构拆解相关部分，必须安排 Manim 画面。
- 开头标题卡、结尾收束卡、每个 `No.x` 或主要制作部分内部的排行卡、关键词爆出、数据计数器、来源信息卡和段落间转场，优先安排 Remotion 画面。
- 真实产品、人物、机构、历史资料、发布会、工厂、实验室，优先安排网上素材。
- 找不到真实素材、需要概念化或微距冲击时，再安排 AI 视频。

每个部分的 8-12 个画面可按内容需要灵活组合，不要套固定比例。常见方向：

- 品牌/销量类 `No.x`：多用网上素材和 Remotion 数据/排行卡，必要时加 Manim 图表。
- 科技原理类 `No.x`：Manim 是核心，搭配真实素材和少量 Remotion 信息卡。
- 历史/发明类 `No.x`：真实历史素材、Manim 时间轴、AI 风格化重建混合使用。
- 诺奖/科学类 `No.x`：官方资料/科学家照片、Manim 原理动画、Remotion 信息卡混合使用。
- 开头部分：优先用 Remotion 标题卡 + 真实/AI 冲击镜头，不加字幕，不要复杂解释。
- 结尾部分：优先用 Remotion 收束卡 + 关键视觉回扣，不重新写口播，不做过长总结。

每个画面要有明确用途：

- `铺真实感`
- `解释抽象概念`
- `打排名/关键词`
- `展示数据`
- `做情绪冲击`
- `转场`

## 网上素材资源

优先顺序：

1. 官方来源：品牌 newsroom、media kit、发布会、公司 YouTube/B 站、年报、投资者演示。
2. 权威资料：NASA、ESA、CERN、NobelPrize.org、大学新闻室、博物馆、专利数据库、Google Patents。
3. 可授权素材库：Pexels、Pixabay、Mixkit、Videvo、Artgrid、Storyblocks、Envato Elements。
4. 开放图片/视频：Wikimedia Commons，必须看具体授权。
5. 中文检索补充：品牌官方微博/公众号、发布会回放、新闻图片、科普机构素材。

输出资源清单时，每条至少包含：

- 用途
- 推荐来源
- 中文搜索词
- 英文搜索词
- 版权提醒

搜索词模板：

```text
[品牌/技术] official newsroom video
[品牌/技术] media kit
[产品名] launch video
[技术名] macro footage
[技术名] animation background
site:nobelprize.org [奖项/技术/科学家]
site:nasa.gov [技术英文名]
Wikimedia Commons [科学家/发明/技术英文名]
[中文技术名] 原理 动画
[中文品牌名] 官方 发布会 素材
```

## Manim 原理动画提示词

Manim 是科技视频的关键画面来源，用来解释真实素材拍不出来的底层机制。适合：

- 芯片晶体管开关
- 光刻光路和掩膜投影
- 锂离子在正负极之间移动
- OLED 像素自发光
- CMOS 把光子转成电信号
- 5G 信号从手机到基站
- GPS 三边定位
- 排行榜柱状图、时间轴、市场份额变化
- 诺奖发现到手机应用的关系链

提示词必须包含：

- 画幅：16:9，1920x1080，30fps
- 时长
- 对应文案原句
- 要解释的机制
- 大致画面框架
- 画面元素和中文标签
- 动画步骤
- 风格约束
- 输出要求

### Manim 统一画面框架

每个 Manim 提示词都要给出大致构图，不要只写“科技感动画”。默认采用下面框架：

```text
画幅：横屏 16:9，1920x1080，30fps。
背景层：纯黑背景，必须使用 #000000 或 Manim BLACK，不使用灰黑渐变，不使用灰色网格铺底。
标题区：画面顶部居中，约占画面高度 10%-14%。只放一个短中文标题，文字加粗，优先白色或高对比强调色。
动画主体区：画面中间，约占画面高度 68%-75%。所有核心图形、公式、粒子、路径、结构拆解都在这里完成。
字幕安全区：画面底部约 14%-18% 保持空白，不放说明文字、不放关键图形，方便后期在剪映加字幕。
安全边距：左右至少保留 5% 空白，重要元素不要贴边。
手机观看：核心图形或关键运动路径要占动画主体区的大部分空间；不要做很多小节点、小公式、小标签堆叠。
```

文字规则：

- 所有中文标题、标签、数字必须加粗；Manim `Text` 使用 `weight=BOLD`，公式优先用 `\mathbf{}` 或粗体样式。
- 禁止使用灰色文字，包括 `GRAY`、`GREY`、`LIGHT_GRAY`、`DARK_GRAY`、`#808080`、`#999999` 等。
- 文字颜色只能用高对比色：白色、亮黄、青色、蓝色、绿色、红色等；正文标签优先白色，重点词用亮黄或青色。
- 不做字幕，不输出整句口播文字；只允许短标签、关键词、数字、公式和标题。
- 中文标题永远比英文更重要；如出现英文，只能作为小号辅助标识。
- 手机可读字号建议：标题 `font_size` 56-76，关键数字 72-110，普通短标签不低于 36；线条和箭头 `stroke_width` 不低于 4。

布局示意：

```text
┌──────────────────────────────────────────────┐
│                 顶部标题区                    │
│          加粗中文标题，居中，不用灰色          │
├──────────────────────────────────────────────┤
│                                              │
│                                              │
│                动画主体区                     │
│      原理结构、路径、粒子、公式、图表变化       │
│                                              │
│                                              │
├──────────────────────────────────────────────┤
│         字幕安全区：保持空白，交给剪映          │
└──────────────────────────────────────────────┘
```

模板：

```text
用 Manim Community Edition 制作一个横屏 16:9 原理动画，1920x1080，30fps，时长 [X] 秒。
用途：解释口播“[粘贴用户原文对应句子]”。
主题：[技术/原理名称]。
视觉风格：纯黑背景，细线条，高对比，蓝白主色，少量亮黄或青色强调，干净高级，不要卡通，不要灰色文字。
画面框架：
1. 顶部标题区：画面顶部居中，放一个加粗中文标题“[短标题]”，不使用灰色。
2. 动画主体区：画面中间展示 [核心结构/机制/图表]，核心运动都在主体区完成。
3. 字幕安全区：画面底部 14%-18% 保持空白，不放任何字幕、说明文字或关键图形，方便剪映后期加字幕。
手机观看可读性：核心元素要足够大，关键主体至少占画面宽度 30%-45%；中文标签不要小于 `font_size=36`，标题建议 `font_size=56-76`，关键数字建议 `font_size=72-110`，线条/箭头 `stroke_width` 不低于 4。
画面元素：[列出对象，例如手机、芯片、晶圆、光源、掩膜、透镜、离子、基站、信号波、坐标轴]。
中文标签：[列出需要出现在画面上的短标签]。
文字要求：所有中文标题、标签、数字必须加粗；禁止使用灰色文字；文字颜色只能用白色、亮黄、青色、蓝色、绿色、红色等高对比色。
动画步骤：
1. [第 0-X 秒：对象如何出现]
2. [机制如何运动或变化]
3. [关键数据/关键词如何弹出]
4. [最后定格在哪个结论画面]
输出要求：无旁白、无背景音乐、无字幕、不要大段文字，背景必须保持纯黑，方便剪映叠加字幕。
代码要求：使用 `from manim import *`；建议设置 `config.background_color = BLACK`；中文 `Text` 使用 `weight=BOLD`；不要使用任何灰色文字颜色常量或灰色十六进制颜色。
```

## Remotion 动效提示词

Remotion 用于包装，不用于替代所有画面。适合：

- 横屏标题开场：必须包含中文主标题和英文副标题
- No.x 排行卡
- 段内标题卡
- 关键词爆出
- 数据计数器
- 柱状图/地图/时间轴包装
- 极简来源角标或下三分之一短标
- 段落转场
- 收尾总结卡

Remotion 的核心不是解释知识点，而是让观众在一晃而过的画面里抓住“排名、数字、关键词、情绪冲击”。不要把 Remotion 做成 PPT 说明页。

文字密度规则：

- 禁止一行一行堆解释性文字，禁止段落说明，禁止长句信息卡。
- 单个 Remotion 镜头最多出现 1 个主标题、1 个数字/排名、1-2 个短关键词。
- 每个中文文本块建议不超过 8-12 个汉字；超过就删成关键词。
- 来源只做极简角标或下三分之一短标，例如 `Source: IDC`、`Apple Newsroom`，不要展开解释。
- 不做字幕，不承接口播全文，不输出整句文案。
- 中文主标题最大，数字/排名第二，英文或来源信息只能小号辅助。
- 手机可读字号建议：中文主标题 96-140px，数字/排名 140-220px，关键词 56-86px，来源角标不低于 28px。

横屏标题开场规则：

- 标题内容取自用户文案里的核心名词，不写口号，不写长句。
- 必须同时给出中文标题和英文标题，例如 `光刻机 / Lithography Machine`、`固态电池 / Solid-State Battery`、`CMOS 图像传感器 / CMOS Image Sensor`。
- 中文标题是主视觉，最大、最清楚、最重要；英文标题作为副标题或背景层辅助。
- 英文标题使用准确的行业常用英文；遇到 OLED、CMOS、GPS、5G 这类英文缩写，可保留缩写并补中文名。
- 中文标题建议 110-160px，英文标题建议 36-56px；两者都要在手机上清晰可读。

动效方向：

- 优先做镜头推近、层级剥离、快速遮罩切入、数字翻滚、排行卡撞入、图形重组、深度视差、粒子扫过、地图/柱状图瞬时生成。
- 每个镜头要有一个明确的视觉动作，不要只是文字淡入淡出。
- 动效要高级、利落、炫酷；避免廉价荧光、满屏霓虹、复杂说明框和过度装饰。
- 观众不暂停也能看懂本镜头的唯一重点。
- 动画主体不要太小：产品图、Logo、数字、排行卡、地图高亮区等关键元素建议占画面宽度 30%-50%。

提示词必须包含：

- 画幅：16:9，1920x1080，30fps
- 时长
- 文案片段
- 视觉风格
- 文本密度限制
- 手机观看可读性
- 层级结构
- 动画节奏
- 素材占位说明

模板：

```text
用 Remotion 制作一个横屏 16:9 动效片段，1920x1080，30fps，时长 [X] 秒。
用途：[横屏标题开场/No.x 排行卡/段内标题卡/关键词爆出/数据计数器/段落转场/来源信息卡/收尾总结卡]。
画面风格：高级科技纪录片，深色背景，强对比中文大字，细线网格，少量扫描光，克制但有冲击力；不要荧光风，不要赛博廉价感。
文本内容：只保留用户原文中的短关键词、数字、排名或品牌名，不改写口播，不放整句解释。
横屏标题开场文本：从文案中提取核心名词，生成 `中文标题：[核心名词中文]` 和 `英文标题：[核心名词英文]`；中文为主标题，英文为副标题。
文本密度：最多 1 个主标题 + 1 个数字/排名 + 1-2 个短关键词；单个中文文本块不超过 8-12 个汉字；禁止多行解释、段落说明和 PPT 式信息卡。
手机观看可读性：中文主标题 96-140px，数字/排名 140-220px，关键词 56-86px，来源角标不低于 28px；关键主体占画面宽度 30%-50%，不要出现小到手机上看不清的 UI、Logo、图表标签或装饰元素。
动画设计：
1. [第 0-X 秒：主体元素如何快速入场，例如撞入、遮罩切入、镜头推近、空间展开]
2. [关键词/数字如何瞬间爆出，停留时间短但识别强]
3. [数据/排名/图形如何用翻滚、重组、扫描、深度视差等方式变化]
4. [如何用一次利落转场进入下一镜]
标题层级：中文标题最大、最重要；英文只能作为小号辅助装饰，不能抢中文主标题。
开头要求：画面简洁，元素逐个出场，节奏要有压迫感和冲击力，但不堆满装饰。
结尾要求：收束干净，保留黑场或暗背景余量，方便剪映添加最后字幕、Logo 或账号信息。
素材位：[需要预留的图片/视频占位，如产品图、芯片图、品牌 Logo]。
输出要求：React/Remotion 可编辑代码，中文文本用常量管理；动画必须由 `useCurrentFrame()` 和 `interpolate()`/`spring()` 驱动，不使用 CSS transition/animation；无字幕、无旁白、无背景音乐。
```

## AI 视频生成提示词

AI 视频用于补足找不到的镜头、抽象概念、微距质感和视觉冲击。可用于可灵、即梦、Runway、Pika、Sora 等工具。不要用 AI 视频伪装真实历史事件或真实人物采访。

适合 AI 生成的镜头：

- 芯片微距、晶圆、光路、数据流
- 电池离子流动的抽象镜头
- 未来感实验室、洁净室、服务器机房
- 手机内部结构概念镜头
- 历史发明的风格化重建
- 技术影响的隐喻画面

提示词必须包含：

- 横屏 16:9
- 时长
- 镜头类型
- 主体
- 运动方式
- 光线和质感
- 禁止项
- 是否需要文字

模板：

```text
生成一个横屏 16:9 科技视频镜头，时长 [X] 秒，写实电影质感，适合科技纪录片。
主体：[具体物体/场景]。
画面：[具体构图和动作]。
镜头运动：[推近/横移/俯拍/微距扫过/慢动作]。
光线和质感：[冷色实验室光/金属质感/暗背景/高对比/微粒/体积光]。
用途：用于口播“[粘贴用户原文对应句子]”。
不要出现：错误品牌 Logo、乱码文字、畸形手、虚假人物采访、夸张卡通风。
输出：无字幕、无旁白、无音乐，方便剪映叠加字幕。
```

## 表格格式

## 版权与事实安全

- 网上素材必须检查版权、商用范围和平台复用许可。
- 官方素材也不等于可随意商用，要看 media usage policy。
- AI 视频不能冒充真实新闻、真实历史影像或真实采访。
- 涉及销量、排名、年份、诺奖、人物和机构时，把文案中的具体事实列入核查点。
- 如果没有合适真实素材，优先写“AI 概念镜头”或“Remotion 信息卡”，不要建议用户硬找错素材。
- 所有图表和数据动效必须来自明确来源；没有来源时标注“数据待核查”。
